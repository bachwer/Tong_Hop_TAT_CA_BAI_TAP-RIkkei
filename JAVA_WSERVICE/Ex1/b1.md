# Bài tập 1

## Câu 1: Trình bày khái niệm Web Service. Cho ví dụ thực tế.

**Khái niệm:** Web Service (Dịch vụ mạng) là một tập hợp các phương thức hoạt động dựa trên môi trường Internet, cho phép hai hoặc nhiều ứng dụng khác nhau (có thể chạy trên các hệ điều hành, viết bằng các ngôn ngữ lập trình khác nhau) giao tiếp, trao đổi dữ liệu và chia sẻ chức năng với nhau một cách tự động.

**Ví dụ thực tế:** Khi bạn vào một trang web đặt vé máy bay (như Traveloka hoặc Agoda), trang web này có thể hiển thị giá vé và chuyến bay của rất nhiều hãng khác nhau (Vietnam Airlines, Vietjet...). Thực chất, Traveloka không tự quản lý dữ liệu này; họ đã gọi các Web Service của các hãng hàng không để lấy thông tin chuyến bay thời gian thực và hiển thị lên cho bạn.

## Câu 2: Hoàn thành bảng so sánh giữa SOAP và REST

Dưới đây là bảng so sánh đã được điền đầy đủ theo đúng 4 tiêu chí yêu cầu:

| Tiêu chí | SOAP | REST |
|----------|------|------|
| Giao thức truyền tải | Bắt buộc sử dụng giao thức chuẩn là SOAP (thường chạy trên nền HTTP, SMTP, FTP...). | Sử dụng trực tiếp các phương thức của giao thức HTTP (GET, POST, PUT, DELETE...). |
| Định dạng dữ liệu chính | Chỉ hỗ trợ duy nhất định dạng XML. | Hỗ trợ rất nhiều định dạng, phổ biến nhất là JSON, ngoài ra có cả XML, Text, HTML... |
| Trạng thái (stateful/stateless) | Có thể cấu hình theo cả hai dạng, nhưng mặc định thường là Stateful (lưu lại trạng thái phiên làm việc). | Bắt buộc là Stateless (mỗi request gửi đi độc lập, server không lưu trạng thái của client). |
| Tính dễ dàng triển khai | Phức tạp, tốn nhiều tài nguyên, cần định nghĩa các file cấu hình nghiêm ngặt (WSDL). | Dễ dàng, gọn nhẹ, trực quan và dễ học hơn rất nhiều so với SOAP. |

## Câu 3: Chuyển đổi dữ liệu từ JSON sang XML tương ứng

**Đoạn JSON gốc:**
```json
{
  "sinhvien": {
    "maso": "SV001",
    "hoten": "Nguyễn Văn A",
    "diem": [8.5, 7.0, 9.0]
  }
}
```

**Đoạn XML tương ứng sau khi chuyển đổi:**
```xml
<sinhvien>
    <maso>SV001</maso>
    <hoten>Nguyễn Văn A</hoten>
    <diem>8.5</diem>
    <diem>7.0</diem>
    <diem>9.0</diem>
</sinhvien>
```


## Câu 4: Nêu ưu điểm của JSON so với XML trong truyền tải dữ liệu qua Web Service

JSON hiện nay là định dạng thống trị trong các hệ thống RESTful API nhờ các ưu điểm vượt trội sau so với XML:

1. **Dung lượng nhẹ hơn, tiết kiệm băng thông:** JSON sử dụng các cặp dấu ngoặc {}, [] để bao bọc dữ liệu, trong khi XML bắt buộc phải có thẻ đóng và thẻ mở (ví dụ: `<maso>SV001</maso>`). Việc loại bỏ các thẻ trùng lặp giúp kích thước file JSON nhỏ hơn rất nhiều so với XML, giúp truyền tải qua mạng nhanh hơn.

2. **Dễ đọc, dễ viết và thân thiện với JavaScript:** Cú pháp của JSON chính là cú pháp tạo đối tượng của ngôn ngữ JavaScript. Do đó, các ứng dụng Frontend (Web, Mobile) có thể dễ dàng chuyển đổi chuỗi JSON thành đối tượng Java/JavaScript để xử lý ngay lập tức thông qua hàm `JSON.parse()`, trong khi xử lý XML phức tạp hơn (cần dùng XML Parser).
