# Bài tập 2

## Câu 1: Liệt kê các thành phần có trong HTTP Request trên và giải thích ý nghĩa từng header

Dựa vào đoạn Request giả định, các thành phần bao gồm:

- **HTTP Method:** POST (Yêu cầu tạo mới một tài nguyên).
- **URL (Path):** /api/sanpham (Đường dẫn nơi nhận yêu cầu).
- **HTTP Version:** HTTP/1.1 (Phiên bản giao thức sử dụng).
- **Request Body:** {"ten":"Laptop","gia":15000000,"tonkho":10} (Dữ liệu sản phẩm mới gửi lên dạng JSON).
- **Request Headers:**
  - **Host:** example.com: Định danh tên miền (domain) của máy chủ mà client đang muốn gửi request tới.
  - **Content-Type:** application/json: Báo cho server biết định dạng dữ liệu nằm trong phần Body gửi lên là kiểu JSON.
  - **Authorization:** Bearer abc123: Chuỗi token (abc123) dùng để xác thực quyền truy cập, chứng minh client đã đăng nhập và có quyền tạo sản phẩm.
  - **Content-Length:** 48: Kích thước (tính bằng byte) của phần dữ liệu trong Body.

## Câu 2: Liệt kê các thành phần có trong HTTP Response. Mã trạng thái 201 Created thuộc nhóm nào và ý nghĩa?

Dựa vào đoạn Response giả định, các thành phần bao gồm:

- **HTTP Version:** HTTP/1.1
- **Status Code & Phrase:** 201 Created
- **Response Body:** {"id":101,"ten":"Laptop","gia":15000000,"tonkho":10} (Dữ liệu của sản phẩm đã được tạo thành công kèm theo ID lưu trong DB).
- **Response Headers:**
  - **Date:** Thời gian phản hồi được tạo ra trên máy chủ (Mon, 10 Apr 2025 07:30:00 GMT).
  - **Content-Type:** Định danh dữ liệu trả về cho client là kiểu JSON.
  - **Location:** Đường dẫn (URL) trực tiếp dẫn đến tài nguyên vừa mới được tạo ra trên hệ thống (/api/sanpham/101).

**Mã trạng thái 201 Created:**
- **Thuộc nhóm:** 2xx (Success - Thành công).
- **Ý nghĩa:** Yêu cầu của client đã thành công và dẫn đến việc một tài nguyên mới đã được tạo ra thành công trên máy chủ.

## Câu 3: Nếu client gửi request GET /api/sanpham/999 nhưng sản phẩm không tồn tại, mã trạng thái HTTP nào sẽ được trả về? Giải thích.

**Mã trạng thái trả về:** 404 Not Found.

**Giải thích:** Đây là mã trạng thái chuẩn trong giao thức HTTP dùng để thông báo rằng máy chủ hoàn toàn chạy bình thường, nhưng không thể tìm thấy tài nguyên ứng với đường dẫn (URL) được yêu cầu. Ở đây, hệ thống không tìm thấy sản phẩm nào có ID là 999.

## Câu 4: Khi server gặp lỗi xử lý không xác định, mã trạng thái nào thuộc nhóm 5xx sẽ được dùng?

**Mã trạng thái trả về:** 500 Internal Server Error.

**Ý nghĩa:** Đây là mã lỗi "bọc lót" phổ biến nhất thuộc nhóm 5xx. Nó được dùng khi máy chủ gặp phải một tình huống/lỗi bất ngờ hệ thống (ví dụ: lỗi crash code, lỗi logic null pointer exception, lỗi database đột ngột...) mà server không biết cách xử lý cụ thể hoặc không có mã lỗi nào khác phù hợp hơn để mô tả lỗi đó.
